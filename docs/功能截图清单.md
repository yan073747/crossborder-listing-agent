# 功能截图清单

本文档用于整理“跨境电商 Listing 优化 Agent”的作品集截图。截图目标不是单纯展示页面，而是证明项目具备完整业务流程、AI Agent 能力、工程实现和可交付价值。

建议最终保留 8-10 张核心截图，放到 GitHub README、简历作品集、面试展示材料中。

## 截图命名规范

建议截图文件统一放在：

```text
docs/screenshots/
```

命名格式：

```text
01-home-product-form.png
02-competitor-csv-import.png
03-llm-generation-result.png
```

文件名使用英文，方便 GitHub 展示；图片标题和说明使用中文，方便面试讲解。

## 必拍截图

| 序号 | 截图名称 | 页面位置 | 要证明的能力 | 面试讲法 |
| --- | --- | --- | --- | --- |
| 1 | 首页完整工作台 | 页面首屏，包含商品表单、语言选择、导出按钮、结果区 | 项目不是单点 Demo，而是一个完整运营工作台 | “我把 Listing 优化流程做成了一个面向运营的工作台，左侧录入商品和竞品，右侧查看生成结果和评分。” |
| 2 | 结构化商品信息录入 | 左侧商品信息录入区域 | 支持业务字段结构化，而不是只输入一段 prompt | “这里把商品名、类目、市场、语言、卖点、规格、价格和品牌语气拆成字段，方便后端稳定处理。” |
| 3 | 生成语言切换 | 顶部生成语言下拉框，或表单内输出语言 | 多语言本地化能力 | “项目默认中文，也支持英语、日语、德语、西班牙语，符合跨境电商多市场运营场景。” |
| 4 | 竞品 CSV 导入 | 竞品 CSV 导入区域和导入后的表格 | 支持竞品数据导入，具备运营分析入口 | “运营可以导入竞品标题、卖点、关键词、价格和评分，Agent 会参考这些信息做定位分析。” |
| 5 | DeepSeek 真实调用结果 | 生成结果顶部显示“大模型生成：deepseek” | 已接入真实大模型，不是纯模板 | “这里可以看到生成来源是 DeepSeek，说明当前结果来自真实 LLM 调用。” |
| 6 | A/B Listing 对比 | 右侧版本 A 和版本 B 卡片 | 支持多版本生成和对比 | “系统一次生成两个版本，并自动推荐更适合上线的版本。” |
| 7 | 评分维度拆解 | 每个版本下方的评分条 | 结果可解释，不只是文本生成 | “我把 Listing 质量拆成标题完整度、关键词覆盖、卖点清晰度、本地化质量和广告转化潜力五个维度。” |
| 8 | 二次优化按钮 | 结果区的 SEO、转化、简洁、本地化优化按钮 | 支持继续迭代，而不是一次性生成 | “如果运营觉得版本还不够好，可以继续按 SEO、转化、简洁或本地化方向优化。” |
| 9 | 历史生成记录 | 历史生成记录区域 | 支持生成记录沉淀和复盘 | “每次生成会自动保存，点击历史记录可以把当次结果加载回来复盘。” |
| 10 | Excel 导出结果 | 点击导出 Excel 后的文件，或浏览器下载记录 + Excel 打开效果 | 项目结果能交付给运营团队 | “最终结果可以导出 Excel，方便运营团队整理、评审和上线。” |

## 推荐截图顺序

如果只放 6 张图，建议使用这个顺序：

1. 首页完整工作台。
2. 商品信息结构化录入 + 竞品 CSV 导入。
3. DeepSeek 生成结果。
4. A/B Listing 对比 + 评分维度。
5. 二次优化 + 历史复盘。
6. Excel 导出结果。

如果放 10 张图，使用“必拍截图”完整顺序。

## 截图质量要求

- 浏览器缩放建议保持 100%。
- 截图宽度建议至少 1280px，避免文字太小。
- 截图中不要露出真实 API Key、系统路径里的隐私信息或无关聊天窗口。
- 每张图只突出一个主题，不要把所有内容都截得很小。
- 结果区截图要包含“生成来源”和“评分维度”，这是 AI Agent 项目的关键证明点。
- 导出截图要能看出 CSV 或 Excel 文件确实生成，而不是只截按钮。
